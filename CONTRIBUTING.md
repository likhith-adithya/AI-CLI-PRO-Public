## Getting Started
AI-CLI-PRO is a **public issue tracker**. The source code is maintained in a private repository for security and development reasons. We value your feedback as the primary driver for our roadmap.

### Prerequisites
*   Ensure you are using the latest version of the extension.
*   Search existing issues to avoid duplicates.
*   Gather necessary environment details (OS, VS Code version).

## How to Report a Bug
### Step 1: Search
Before reporting, [search the issues](../../issues) to see if the bug is already documented.

### Step 2: Create a Bug Report
1. Go to [Issues](../../issues) -> **New Issue** -> **Bug Report**.
2. Provide a clear, concise title.
3. Fill in the template with:
    *   **Steps to Reproduce:** Exactly how to trigger the bug.
    *   **Expected Behavior:** What should have happened.
    *   **Actual Behavior:** What actually happened.
    *   **Environment Details:** OS, VS Code Version, Extension Version.

## How to Request a Feature
### Step 1: Check for Duplicates
[Search feature requests](../../issues?q=is%3Aissue+label%3A%22feature-request%22) to see if your idea has been suggested.

### Step 2: Submit a Request
1. Go to [Issues](../../issues) -> **New Issue** -> **Feature Request**.
2. Explain the **Problem Statement**: What challenge are you facing?
3. Describe the **Proposed Solution**: How should the feature work?
4. Mention the **Use Case**: Why is this important to you?

## Issue Quality Guidelines
✅ **DO:**
*   Use descriptive titles.
*   Attach screenshots or logs for clarity.
*   Be respectful and constructive.

❌ **DON'T:**
*   Post duplicate issues.
*   Share sensitive data (API keys, passwords).
*   Use a demanding or aggressive tone.
